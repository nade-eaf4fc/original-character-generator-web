# Contributing

Thanks for your interest in improving this project.

## Before You Start

- Please check the README first for the app structure and feature overview.
- If you are planning a larger feature or UI change, opening an issue first is helpful.
- Keep changes focused and easy to review.

## Development Notes

- The app entry point is `index.html`.
- Main logic lives in `scripts/app.js`.
- Styling lives in `styles/site.css`.
- Data files live under `data/`.

## Pull Requests

- Use clear commit messages.
- Explain what changed and why.
- Update the README if behavior or setup changes.
- For visual changes, screenshots are very helpful.

## Scope

Good contribution targets include:

- bug fixes
- documentation updates
- accessibility improvements
- UI polish
- data updates
- ComfyUI integration improvements

## Be Careful With

- breaking existing `settings_json` compatibility
- introducing private data, local paths, or personal files
- changing generated output behavior without documenting it
